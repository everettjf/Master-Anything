# Master-Anything · 愿景 (Vision)

> **Master anything, verifiably.** 不只让你看懂,而是练到会、并且能证明你会。

## 1. 一句话定位

Master-Anything 把**任何知识**(首先是代码仓库)转化为一张知识图谱,并在其上叠加一个
**精通引擎**:它为每个学习者建模"掌握度",生成自适应的学习路径、测验与练习,
并尽可能用**可验证**的方式判定"你是否真的掌握了"。

## 2. 我们和 "Understand-Anything" 的根本区别

业界已有的同类工具(如 Understand-Anything)解决的是 **Understand(理解)**:
一次性把代码扫描成知识图谱 + dashboard,帮你"看懂"。

但"理解"是内容的一次性快照,"精通"是**学习者的一个持续状态**。这带来一个关键洞察:

> **不同领域只是"输入"不同;一旦内容变成知识图谱,"如何让人掌握它"是同一套引擎。**

| 维度 | Understand 类工具 | Master-Anything |
|---|---|---|
| 目标 | 看懂(read-only 可视化) | 练会(掌握度状态) |
| 产出 | 一张图 / dashboard | 自适应学习路径 + 可验证精通 |
| 数据资产 | 内容图谱 | 内容图谱 **+ 每个学习者的掌握图谱** |
| 时效 | 快照,易过时 | 随内容增量更新,随学习持续演进 |
| 领域 | 多为单一(代码) | 可插拔适配器,真正面向 "Anything" |

## 3. 三条核心原则

### 3.1 精通是领域无关的 (Mastery is domain-agnostic)
领域适配层各不相同,但"知识图谱 → 拆解 → 建模 → 评估"的精通引擎只写一次。
这是支持 "Anything" 而不让系统爆炸的底层逻辑。

### 3.2 精通要可验证 (Verifiable Mastery)
- 通用知识:生成式测验 + AI 评分(基于源头 grounding,减少幻觉)。
- **代码:可以让学习者真的改一段代码 / 写一个测试,用真实测试是否通过来客观判定。**

代码是唯一拥有 ground truth 的领域,能让"精通"变得**可证伪**。这正是我们优先做代码的
战略理由:先在代码场景树立"可验证精通"的标杆,再迁移到更软的知识场景。

### 3.3 掌握度会沉淀 (Mastery compounds)
每个用户的掌握图谱随时间积累,越用越准、越个性化。**护城河是学习者数据,而非内容本身。**

## 4. 精通模型:Bloom 分级

每个知识节点不是"懂/不懂"的二元状态,而是沿 Bloom 认知层级推进:

```
记忆 (Remember) → 理解 (Understand) → 应用 (Apply) → 分析 (Analyze) → 创造 (Create)
```

系统始终知道学习者在每个节点处于哪一级,并据此决定下一步该复习、该讲解、还是该出练习。

## 5. "Anything" 的扩展次序

精通引擎与图谱 schema 保持不动,每个阶段只新增一个**领域适配器**:

| 阶段 | 领域 | 战略意义 |
|---|---|---|
| **P0** | 📦 代码仓库 | 可验证精通的标杆;复用成熟的 tree-sitter + LLM 方案 |
| **P1** | 📄 文档 / PDF / 书籍 | 适配器最简单,首次证明"通用引擎真的通用" |
| **P2** | 📚 论文 / 技术博客 | 引入引用图谱,验证 schema 扩展性 |
| **P3** | 🎥 课程 / 视频(转写) | 切入教育市场,内容量大 |
| **P4** | 🌐 任意网页 / 个人笔记库 | 真正的 "Master Anything",个人知识管理 |

> **里程碑判定**:当 P1 跑通时,"换个适配器就能掌握新领域"被证明,"Anything" 命题成立。

## 6. 产品形态

长期为 **Web + IDE/CLI 双形态**,二者共享同一套图谱与精通引擎后端:

- **先行(Web)**:独立 Web 应用,连接仓库 / 上传内容,在浏览器内完成"学—测—练"闭环,
  最快验证精通闭环与留存。
- **跟进(IDE / CLI)**:类似 `/understand` 的命令,集成进 Claude Code / Cursor 等开发者工作流,
  让"在写代码的地方就能精通这段代码"。

## 7. 北极星指标

不是"生成了多少图谱",而是 **"用户在多少知识节点上达到了可验证的高阶掌握(应用级及以上)"**。
一切功能都服务于推高这个数字。
